# ប្រព័ន្ធគ្រប់គ្រងហត្ថលេខា (Signature Management System)

## សង្ខេបមុខងារ

ប្រព័ន្ធគ្រប់គ្រងហត្ថលេខាជាផ្នែកមួយនៃប្រព័ន្ធគ្រប់គ្រងឯកសារ ដែលអនុញ្ញាតឱ្យអ្នកប្រើប្រាស់អាចរក្សាទុក គ្រប់គ្រង និងប្រើប្រាស់ហត្ថលេខាដោយស្វ័យប្រវត្តិនៅក្នុងការបែបបទឯកសារ។

## លក្ខណៈពិសេស

### 1. **គ្រប់គ្រងហត្ថលេខា**
- បង្កើតហត្ថលេខាថ្មី
- កែប្រែហត្ថលេខាដែលមានស្រាប់
- លុបហត្ថលេខា
- ស្វែងរកហត្ថលេខាតាមឈ្មោះ ឬលេខកូដ

### 2. **ការចាត់ថ្នាក់**
- **ឈ្មោះខ្មែរ**: ឈ្មោះជាភាសាខ្មែរ (ឧ. នាយកការិយាល័យ, នាយករង១)
- **លេខកូដបុគ្គលិក**: លេខកូដបុគ្គលិក (ឧ. D001, E0001, M0001)
- **នាយក**: ហត្ថលេខានាយកមន្ទីរពេទ្យ
- **នាយករង**: ហត្ថលេខានាយករងផ្សេងៗ
- **ការិយាល័យ**: ហត្ថលេខាស្ថាប័នការិយាល័យ
- **រដ្ឋបាល**: ហត្ថលេខាបុគ្គលិករដ្ឋបាល

### 3. **ការស្វែងរកដោយស្វ័យប្រវត្តិ**
- ការស្វែងរកមិនប្រកាន់ case-sensitive
- ទាមទារការផ្គូផ្គង 100% សម្រាប់ឈ្មោះ ឬលេខកូដ
- ការចុះកំណត់ត្រាការប្រើប្រាស់ដោយស្វ័យប្រវត្តិ

### 4. **ព័ត៌មានលម្អិត**
- ឈ្មោះពេញជាខ្មែរ
- តួនាទី/មុខតំណែង
- ផ្នែក/នាយកដ្ឋាន
- ការពិពណ៌នា
- ស្ថានភាព (សកម្ម, អសកម្ម, បណ្ណសារ)
- កាលបរិច្ឆេទផុតកំណត់
- កំណត់សម្គាល់

### 5. **ស្ថិតិនិងតាមដាន**
- ចំនួនដងប្រើប្រាស់
- ថ្ងៃខែឆ្នាំចុងក្រោយដែលបានប្រើ
- ស្ថិតិតាមប្រភេទ
- ស្ថិតិសរុប

## API Endpoints

### GET /api/signatures
ទទួលបានបញ្ជីហត្ថលេខាទាំងអស់

**Parameters:**
- `type` (optional): ការត្រងតាមប្រភេទ
- `status` (optional): ការត្រងតាមស្ថានភាព
- `search` (optional): ការស្វែងរកក្នុងឈ្មោះ, តួនាទី, ផ្នែក
- `page` (optional): ទំព័រសម្រាប់ pagination
- `limit` (optional): ចំនួនលទ្ធផលក្នុងមួយទំព័រ

### GET /api/signatures/search/:name
ស្វែងរកហត្ថលេខាតាមឈ្មោះពិតប្រាកដ

**Response:** ហត្ថលេខាដែលត្រូវគ្នា ឬ 404 បើរកមិនឃើញ

### GET /api/signatures/type/:type
ទទួលបានហត្ថលេខាតាមប្រភេទ

**Types:** khmer, employee, director, deputy, office, admin

### GET /api/signatures/:id
ទទួលបានហត្ថលេខាតាម ID

### POST /api/signatures
បង្កើតហត្ថលេខាថ្មី

**Body:**
- `name` (required): ឈ្មោះ ឬ លេខកូដ
- `fullNameKh` (optional): ឈ្មោះពេញជាខ្មែរ
- `type` (optional): ប្រភេទ (default: employee)
- `description` (optional): ការពិពណ៌នា
- `position` (optional): តួនាទី
- `department` (optional): ផ្នែក
- `notes` (optional): កំណត់សម្គាល់
- `expiryDate` (optional): កាលបរិច្ឆេទផុតកំណត់
- `signatureFile` (optional): ឯកសារហត្ថលេខា
- `filePath` (optional): ផ្លូវឯកសារ (បើមិនបានបញ្ចូលឯកសារ)

### PUT /api/signatures/:id
កែប្រែហត្ថលេខា

**Body:** ដូចនឹង POST តែ fields ទាំងអស់ optional

### DELETE /api/signatures/:id
លុបហត្ថលេខា

### GET /api/signatures/stats/overview
ទទួលបានស្ថិតិសរុប

## ការប្រើប្រាស់

### 1. **ការដំឡើងនិងការចាប់ផ្តើម**

```bash
# បញ្ចូលទិន្នន័យសាកល្បង
cd backend
npm run seed:signatures

# ធ្វើតេស្តប្រព័ន្ធ
npm run test:signatures

# ចាប់ផ្តើម server
npm run dev
```

### 2. **ការចូលប្រើ**
1. ចូលទៅ ឯកសារ > គ្រប់គ្រងហត្ថលេខា
2. ប្រើប្រាស់ពុម្ពស្វែងរក និងការត្រង
3. ចុច "បន្ថែមហត្ថលេខាថ្មី" ដើម្បីបង្កើត
4. ចុចលើ "កែប្រែ" ដើម្បីកែប្រែ
5. ចុចលើ "ទេស្ត" ដើម្បីពិនិត្យដំណើរការស្វែងរក

### 3. **ការប្រើប្រាស់ក្នុង Document Flow**
នៅពេលបញ្ចូលឈ្មោះក្នុងការបែបបទឯកសារ:
1. បញ្ចូលឈ្មោះ ឬលេខកូដត្រឹមត្រូវ
2. ហត្ថលេខានឹងបង្ហាញដោយស្វ័យប្រវត្តិ
3. ការប្រើប្រាស់នឹងត្រូវបានកត់ត្រា

## ឧទាហរណ៍ទិន្នន័យ

ប្រព័ន្ធរួមបញ្ចូលទិន្នន័យសាកល្បងដូចជា:

**ហត្ថលេខាភាសាខ្មែរ:**
- នាយកការិយាល័យ
- នាយករង១
- នាយករង២
- នាយកមន្ទីរពេទ្យ

**លេខកូដបុគ្គលិក:**
- D001, D002, D007, D009, D0001, D0002, D0007, D0009, D00011, D0015, D0019, D0020, D0022
- E0001, E0002 (ផ្នែកអាសន្ន)
- M0001, M0002 (ផ្នែកសម្ភព)
- A0001 (រដ្ឋបាល)

## ការកំណត់ Permissions

ហត្ថលេខាត្រូវការ permissions ដូចខាងក្រោម:
- `canViewDocuments`: មើលបញ្ជីហត្ថលេខា
- `canEditDocuments`: បង្កើត/កែប្រែ/លុបហត្ថលេខា

## បញ្ហាទូទៅ

### ហត្ថលេខាមិនបង្ហាញ
1. ពិនិត្យថាឈ្មោះ ឬលេខកូដបានបញ្ចូលត្រឹមត្រូវ
2. ពិនិត្យថាឯកសារមាននៅក្នុងថត /public/Uploads
3. ពិនិត្យថាស្ថានភាពជា "active"

### ឯកសារមិនអាចបង្ហាញ
1. ពិនិត្យ file path ត្រឹមត្រូវ
2. ពិនិត្យ file permissions
3. ពិនិត្យថាឯកសារជា image format ដែលគាំទ្រ

### Performance Issues
1. ប្រើ pagination សម្រាប់បញ្ជីធំ
2. ប្រើការត្រងដើម្បីកំណត់លទ្ធផល
3. ពិនិត្យ MongoDB indexes

## ការអភិវឌ្ឍន៍បន្តបន្ទាប់

1. **Image Processing**: ការកាត់តូច resize ស្វ័យប្រវត្តិ
2. **Bulk Upload**: ការបញ្ចូលហត្ថលេខាជាច្រើនដដែលគ្នា
3. **Digital Signatures**: ការគាំទ្រ digital signature
4. **Audit Trail**: ការតាមដានការផ្លាស់ប្តូរលម្អិត
5. **Integration**: ការតភ្ជាប់ជាមួយ HR system
6. **Mobile Support**: ការគាំទ្រសម្រាប់ mobile devices
